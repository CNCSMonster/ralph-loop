# Ralph Loop

极简自动化开发循环 - 只需一个 `PROMPT.md`，让 AI Agent 自动完成开发任务。

## 安装

单脚本，无需依赖，复制到 `~/.local/bin` 即可全局使用：

```bash
cp ralph-loop ~/.local/bin/ralph-loop
chmod +x ~/.local/bin/ralph-loop
```

确保 `~/.local/bin` 在你的 `PATH` 中：

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

安装后任意目录可直接运行 `ralph-loop` 命令。

## 原理

核心就是一个 `while` 循环，每次全新上下文启动 AI，通过文件系统持久化状态。

## 快速开始

### 1. 编写需求

创建 `PROMPT.md`，写入你的项目需求。

### 2. 初始化

```bash
ralph-loop init
```

自动生成 `.ralph/` 下的约束文件和状态文件模板（不会覆盖已有文件）。

### 3. 启动循环

```bash
ralph-loop run        # 默认 50 次迭代
ralph-loop run 100    # 指定 100 次
```

### 4. 配合 AI

脚本会自动启动 `qwen --yolo` 执行任务，无需人工干预。

如果需要在迭代之间暂停观察，可以在 qwen 执行完成后按 `Ctrl+C` 中断脚本。

### 5. 断点续跑

如果迭代次数用完但任务没完成，直接再次 `run` 即可，会从上次进度继续：

```bash
ralph-loop run 20     # 继续之前的进度
```

## 命令

| 命令 | 说明 |
|------|------|
| `ralph-loop init` | 初始化项目结构（幂等，不覆盖已有文件） |
| `ralph-loop run [N]` | 启动循环，N 为最大迭代次数（默认 50） |
| `ralph-loop status` | 生成状态分析 prompt，交给 qwen-code 分析 |
| `ralph-loop help` | 显示帮助 |

## 文件结构

```
.
├── PROMPT.md                     # 需求文档（唯一需要你写的）
├── ralph-loop                    # 主脚本
└── .ralph/
    ├── constraints.md            # AI 约束规则（自动生成，❌ 禁止修改）
    ├── plan.md                   # 计划与任务状态（✅ AI 更新）
    ├── progress.md               # 进度记录（✅ AI 更新）
    ├── verify.md                 # 验证记录（✅ AI 更新）
    └── docs/                     # 子任务 spec（✅ AI 可选）
```

## 工作流程

```
PROMPT.md
  ↓
ralph-loop run
  ↓
while 循环 N 次:
  1. AI 读取 constraints.md + plan.md + progress.md
  2. AI 判断是否需要调整 plan.md
  3. AI 选择下一个任务 → 实现 → 验证
  4. AI 更新 plan.md + progress.md + verify.md
  5. 脚本规则检查:
     - 验证发现问题 → 次数归零 → 继续
     - 全部通过 或 验证3次 → 结束
     - 其他 → 继续
```

## PROMPT.md 编写规范

`PROMPT.md` 是你与 AI Agent 沟通的唯一入口，写好它至关重要。

### 核心原则

1. **明确目标**：用一句话说明项目要做什么
2. **描述当前状态**：已有代码/文件结构，哪些已完成哪些待做
3. **列出具体要求**：功能点、技术栈、约束条件
4. **定义验证方式**：如何判断任务完成（测试命令、手动检查项等）
5. **提供上下文**：相关背景信息、参考文档、已知问题

### 推荐结构

```markdown
# 项目名称

## 目标
一句话描述项目要做什么。

## 当前状态
- 已有：列出已有文件/功能
- 待完成：列出需要实现的功能
- 已知问题：列出已知 bug 或限制

## 具体要求
1. 功能需求 1
2. 功能需求 2
3. 技术约束（如：不引入外部依赖、使用 Python 3.10+ 等）

## 验证方式
- 命令 1 → 期望输出/行为
- 命令 2 → 期望输出/行为
- 其他检查项

## 提示
- 任何有助于 AI 理解项目的信息（如相关文档链接、设计决策原因等）
```

### 示例：CLI 工具

```markdown
# 待办事项 CLI

## 目标
创建一个简单的命令行待办事项管理工具。

## 当前状态
- 空项目，只有 README.md
- 需要从零开始实现

## 具体要求
1. 支持添加/完成/删除/列出待办事项
2. 数据持久化到 JSON 文件
3. 使用 Python 编写，不引入第三方依赖
4. 每个命令有清晰的输出格式

## 验证方式
- `todo add "买牛奶"` → 输出 "已添加: 买牛奶"
- `todo list` → 列出所有未完成事项
- `todo done 1` → 标记第 1 项为完成
- `todo list --all` → 列出所有事项（含已完成）

## 提示
- 数据存储格式建议：`{"items": [{"id": 1, "text": "...", "done": false}]}`
- 命令使用 argparse 解析
```

### 示例：Web 应用

```markdown
# 个人博客

## 目标
使用 Flask 构建一个简单的个人博客系统。

## 当前状态
- 已有基础的 app.py 和 templates/ 目录
- 数据库模型已定义（Post 表）
- 缺少首页、文章详情、创建文章功能

## 具体要求
1. 首页显示文章列表（按时间倒序）
2. 点击文章标题进入详情页
3. 支持 Markdown 格式渲染
4. 管理员可创建/编辑文章（简单密码保护）
5. 使用 SQLite 存储数据

## 验证方式
- 访问 `/` → 显示文章列表
- 访问 `/post/1` → 显示文章详情
- 访问 `/admin` 输入密码后 → 显示管理界面
- 创建文章后刷新首页 → 新文章出现在列表

## 提示
- Markdown 渲染可使用 `markdown` 库（已安装在 requirements.txt）
- 密码保护用简单的 session 机制即可，不需要完整认证系统
```

### 避免的写法

❌ **太模糊**："做一个好用的工具"
✅ **明确**："创建一个支持自动补全的命令行文件搜索工具"

❌ **缺少验证标准**："优化性能"
✅ **可测量**："将启动时间从 2s 降低到 500ms 以内"

❌ **假设 AI 知道上下文**："把之前的 bug 修一下"
✅ **提供上下文**："修复 status 命令统计 bug：init 后显示总任务数为 2，实际 plan.md 中没有任务定义"

## 约束机制

`.ralph/constraints.md` 定义 AI 必须遵守的规则：
- 调整计划 → 选择任务 → 实现 → 验证 → 更新状态 → 提交
- **只有此文件禁止 AI 修改**，其他状态文件 AI 自由读写
- `plan.md` 有格式约定，脚本通过正则 `passes:(true|false)` 统计任务状态

## 完成条件

循环在以下情况结束：
- ✅ `.ralph/plan.md` 中所有任务 `passes:true`
- ✅ `.ralph/verify.md` 验证次数达到 3 次
- ❌ 达到最大迭代次数
- 用户输入 `quit`

## 常见问题

**Q: 可以中断后继续吗？**
A: 可以。状态持久化在文件中，随时 `ralph-loop run N` 即可续跑。

**Q: 会覆盖我之前写的进度吗？**
A: 不会。`init` 是幂等的，已有文件不会覆盖。

**Q: 如何自定义验证命令？**
A: 在 `PROMPT.md` 中说明项目使用的测试/验证方式，AI 会按照约束规则执行。
