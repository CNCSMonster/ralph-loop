# Ralph Loop 约束文件

**警告：此文件定义了 Ralph Loop 的工作规则，AI Agent 必须严格遵守，不得修改此文件。**

## 核心规则

### 1. 工作流程

每次启动时，你必须按以下顺序执行：

```
1. 读取 PROMPT.md（需求文档）
2. 读取 .ralph/constraints.md（约束规则，禁止修改）
3. 读取 .ralph/plan.md（当前计划）
4. 判断计划是否需要调整：
   - 如果需要 → 调整 .ralph/plan.md
   - 如果不需要 → 选择下一个任务
5. 从 plan.md 中选择下一个未完成的高优先级任务
6. 实现该任务
7. 运行测试/验证（仅调试，不累加验证次数）
8. 更新 .ralph/plan.md（标记任务 passes:true/false）
9. 更新 .ralph/progress.md（记录本次迭代做了什么）
10. 如果所有任务都 passes:true，进入验证阶段：
    a. 运行完整验证/测试命令
    b. 更新 .ralph/verify.md（累加验证次数 或 归零）
    c. 如果验证失败，返回步骤 4 继续实现
    d. 如果验证通过 3 次，输出 <promise>COMPLETE</promise>
11. 提交代码变更
```

### 2. 状态文件管理

**.ralph/plan.md** - 计划与任务状态（核心文件）
- 每次必须读取此文件
- 首先判断计划是否需要调整，需要则修改
- 任务状态通过行内标记追踪，格式：
  `- [ ] task-001 任务描述 (priority:1, passes:false)`
  `- [x] task-002 已完成 (priority:2, passes:true)`
- `[ ]` = 未完成，`[x]` = 完成
- `passes:false/true` = 验证是否通过
- 如果需要写子任务 spec，存入 `.ralph/docs/` 目录

**.ralph/progress.md** - 进度记忆
- 每次迭代追加记录
- 包含：本次做了什么、发现了什么、学到了什么

**.ralph/verify.md** - 验证记录
- 记录验证次数和结果
- 每次验证后更新验证次数

### 3. plan.md 格式约定

你必须遵守以下格式，以便脚本能解析任务状态：

```markdown
# 工作计划

## 需求分析
简要分析需求...

## 任务列表
- [ ] task-001 用户登录 (priority:1, passes:false)
- [x] task-002 数据库 (priority:2, passes:true)

## 调整历史
...
```

**关键约定**：
- 任务行必须以 `- [ ]` 或 `- [x]` 开头
- 任务 ID 格式为 `task-NNN`
- 括号内必须包含 `priority:N` 和 `passes:true/false`
- 脚本通过正则 `passes:(true|false)` 统计状态

### 4. 任务选择策略

- 从 `.ralph/plan.md` 中选择 `passes:false` 且 `[ ]` 的任务
- 优先选择 `priority` 数值最小的（优先级最高）
- 如果多个任务优先级相同，选择 ID 最小的

### 5. 验证规则

**验证阶段与实现阶段分离**：
- **实现阶段**：从 plan.md 中选择未完成的任务，逐个实现，每次只做一个任务
- **验证阶段**：只有当 plan.md 中**所有任务**都标记为 `passes:true` 时，才进入验证阶段
- 在实现阶段，**不得累加验证次数**，只做功能测试和调试
- 进入验证阶段后，每次完整运行项目的测试/验证命令，记录结果到 `.ralph/verify.md`

**验证通过时**：
```
验证通过: yes
发现问题: 无
```
并将验证次数 +1（修改 `验证次数: N/3` 中的 N）。

**验证失败时**：
```
验证通过: no
发现问题: {具体问题描述}
```
并将验证次数**归零**（修改 `验证次数: N/3` 为 `验证次数: 0/3`），同时将至少一个任务标记回 `passes:false`，返回实现阶段继续修复。

**你需要自行判断何时调整计划**：
- 如果验证失败，你需要分析问题，在 `.ralph/plan.md` 中调整后续计划
- 脚本不会替你决定是否需要调整计划，这是你的职责

### 6. 完成信号

当满足以下条件时，输出 `<promise>COMPLETE</promise>`：
- `.ralph/plan.md` 中所有任务 `passes:true` **且** `.ralph/verify.md` 中验证次数达到 3 次

注意：仅所有任务 passes:true 是不够的，必须验证阶段也通过 3 次才算完成。

### 7. 代码提交

- 每次迭代完成后，必须提交代码变更
- 提交信息格式：`[ralph-loop] 完成任务: {任务标题} (迭代 #{N})`

## 禁止行为

- ❌ 不得修改 `.ralph/constraints.md`
- ❌ 不得跳过状态文件的读取和更新
- ❌ 不得在一次迭代中尝试完成多个任务
- ❌ 不得在没有运行验证的情况下标记任务为 passes:true
- ❌ 不得忽略验证次数的计数
- ❌ 不得违反 plan.md 格式约定（否则脚本无法统计）
