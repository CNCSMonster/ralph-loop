# 进度记录

## 迭代日志

### 迭代 1 - 修复 status 命令统计 bug
**做了什么**：
- 修复了 `get_task_stats` 函数的正则表达式，现在只匹配以 `- [ ]` 或 `- [x]` 开头的实际任务行
- 修改了 `init` 命令生成的 plan.md 模板，移除注释块中的示例任务行，避免被误匹配
- 修复了 `grep -c` 在无匹配时返回退出码导致变量赋值失败的问题

**发现了什么**：
- bug 根源：plan.md 模板的注释块中包含示例任务行 `- [ ] task-001 ... (passes:false)` 和 `- [x] task-002 ... (passes:true)`
- `grep -c 'passes:true'` 会匹配文件中所有包含该字符串的行，包括注释和约束文件中的示例
- bash 中 `grep -c` 在无匹配时返回退出码 1，需要 `|| true` 来处理

**学到了什么**：
- 统计任务时必须使用更精确的正则，只匹配实际任务行
- 模板中的示例不应该使用与真实数据相同的格式，以免被误统计
